# Initial Requirement Set

Source artifact: [`../artifacts/requirements/initial Requirement Set .docx`](../artifacts/requirements/initial%20Requirement%20Set%20.docx)

## Functional requirements (initial)

| ID | Requirement |
|----|-------------|
| FR-1 | Allow users to create a note with a title and content. |
| FR-2 | Allow users to view all saved notes. |
| FR-3 | Allow users to retrieve a specific note by title or identifier. |
| FR-4 | Allow users to update an existing note. |
| FR-5 | Allow users to delete a note. |
| FR-6 | Save notes in local file storage and load them when the application starts. |

## Non-functional requirements (initial)

| ID | Requirement |
|----|-------------|
| NFR-1 | Respond to create, update, delete, and retrieve without noticeable delay during normal use. |
| NFR-2 | Provide clear success and error messages for user actions. |
| NFR-3 | Validate note input before saving data. |
| NFR-4 | Handle file storage errors gracefully without crashing. |
| NFR-5 | Protect saved notes from accidental data loss during save or update operations. |

These initial requirements were refined to resolve ambiguity and edge cases. See [refined-requirements.md](./refined-requirements.md).
