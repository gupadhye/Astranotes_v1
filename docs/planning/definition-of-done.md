# Definition of Done

Source artifact: [`../artifacts/planning/Part 2 Definition of Done .docx`](../artifacts/planning/Part%202%20Definition%20of%20Done%20.docx)

## Criteria

A feature is **done** when:

1. Work directly supports the refined requirements.
2. The implementer can explain the logic and design decisions.
3. Output is reviewed for correctness and consistency with AstraNotes architecture.
4. The feature is tested with realistic scenarios (unit tests and manual desktop verification).
5. Input validation is in place where required.
6. Error handling prevents crashes and shows clear user-facing messages.
7. Success and failure can be demonstrated (tests or UI feedback).
8. Implementation is straightforward and maintainable.
9. No major known issues are ignored.
10. Work is organized and traceable to requirements.
11. Traceability to requirements and UML is documented.

## Reflection

Definition of Done ensures AstraNotes work is **validated**, not merely built. Each feature must be tested and demonstrated. Without explicit validation criteria, AI-assisted output can appear complete while missing edge cases. Testing and system understanding are required before a feature is accepted.
