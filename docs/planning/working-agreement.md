# Working Agreement

Source artifact: [`../artifacts/planning/Part 1 Working Agreement.docx`](../artifacts/planning/Part%201%20Working%20Agreement.docx)

## Team practices

- Break the project into separate tasks by feature (create, list, search, update, delete, persistence, security).
- Maintain a backlog and select work each sprint.
- Use AI to generate ideas, suggest improvements, and identify edge cases.
- Treat AI output as a **draft**, not a final solution.
- Review prompts and outputs before implementation.
- Ensure decisions (error handling, validation, storage) are understood before moving forward.
- Re-check AI-generated output after changes to confirm alignment with requirements.
- Stay focused on current requirements; avoid gold-plating.
- Avoid conflicting AI outputs without comparison.
- Prioritize simple, maintainable solutions.
- Test features with realistic scenarios.
- Review previous work to avoid rebuilding or inconsistent patterns.

## Reflection

The working agreement defines a structured approach for AstraNotes: plan tasks, use AI orchestrated (not ad hoc), and validate before handoff. Breaking work into smaller tasks and validating outputs reduces errors and wasted effort. A regular review checkpoint before starting new tasks helps ensure work meets the required standard.
